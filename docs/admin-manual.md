# 매듭 품질 검사 서비스 관리자 매뉴얼

이 문서는 관리자가 서비스를 운영하는 방법을 안내합니다.

## 1. 관리자 접속 정보

- 관리자 로그인: `/admin/login`
- 관리자 홈: `/admin`
- 관리자 아이디: `admin`
- 관리자 비밀번호: `admin1234!`

## 2. 로그인 방법

1. `/admin/login`으로 이동합니다.
2. 관리자 아이디와 비밀번호를 입력합니다.
3. `관리자 로그인` 버튼을 누릅니다.
4. 로그인 성공 시 관리자 홈(`/admin`)으로 이동합니다.

## 3. 관리자 홈

관리자 홈에서는 다음 정보를 확인할 수 있습니다.

- 오늘 총 검사 수
- 합격률
- 평균 정확도
- 최근 검사 이력

또한 아래 관리 메뉴로 이동할 수 있습니다.

- 작업자 관리
- 목표 관리
- 공지 관리
- 신고 관리
- 가이드 관리

## 4. 작업자 관리

작업자 관리 화면(`/admin/workers`)에서 할 수 있는 일:

- 작업자 등록
- 작업자 목록 조회
- 작업자 수정
- 작업자 비활성화 처리

등록 및 수정 항목:

- 이름
- 연락처
- 담당 매듭 종류
- 특이사항
- 활성 여부

상세 화면(`/admin/workers/[id]`)에서는 아래 통계를 확인할 수 있습니다.

- 누적 생산량
- 합격률
- 합격 건수
- 불합격 건수

## 5. 목표 생산량 관리

목표 관리 화면(`/admin/goals`)에서 작업자별 일일 목표를 관리합니다.

- 작업자별 오늘 목표 개수 설정
- 현재 달성 수 확인
- 달성률 확인
- 최근 목표 이력 확인

추가 안내:

- 메인 검사 화면에서 합격이 발생하면 달성 수가 자동으로 증가합니다.

## 6. 공지 관리

공지 관리 화면(`/admin/notices`)에서 공지를 등록하고 확인할 수 있습니다.

- 공지 제목 입력
- 공지 본문 입력
- 공지 등록
- 등록된 공지 목록 확인

작업자는 `/notices`에서 공지를 확인합니다.

## 7. 신고 관리

신고 관리 화면(`/admin/reports`)에서 작업자 신고를 확인하고 처리합니다.

- 접수된 신고 목록 확인
- 신고 유형 확인
- 처리 상태 확인
- 상태를 `미처리`에서 `처리완료`로 변경

## 8. 매듭 가이드 관리

가이드 관리 화면(`/admin/guides`)에서 매듭별 단계 가이드를 관리합니다.

- 매듭 종류 선택
- 단계 번호 입력
- 이미지 업로드
- 설명 입력
- 등록 및 수정

작업자는 `/guides`와 `/guide/[knotType]` 화면에서 가이드를 확인합니다.

## 9. 출퇴근 관리자 화면

출퇴근 관리자 화면(`/attendance/admin`)에서 다음 내용을 확인할 수 있습니다.

- 오늘 기록 수
- 이번 달 총 근무 시간
- 기록된 작업자 수
- 월별 근무 시간 합산 표
- 최근 출퇴근 기록

## 10. 권한 안내

- 관리자 로그인 전에는 관리자 전용 메뉴가 보이지 않습니다.
- 로그인하지 않은 상태에서 `/admin`, `/admin/guides`, `/admin/reports`, `/admin/workers` 같은 주소를 직접 입력해도 관리자 로그인 화면으로 이동합니다.
- 작업자용 화면인 `/guides`, `/reports`, `/attendance`, `/notices`는 로그인 없이 사용할 수 있습니다.

## 11. 자주 묻는 질문

### Q1. 관리자 화면이 열리지 않습니다.

다음 항목을 확인합니다.

- 관리자 로그인 여부
- 아이디와 비밀번호 입력 여부
- 세션이 만료되었는지 여부

### Q2. 작업자가 목록에 나오지 않습니다.

해당 작업자가 비활성화되었는지 확인합니다.

### Q3. 출퇴근 기록이나 작업자 정보가 저장되지 않습니다.

다음 항목을 확인합니다.

- DB 연결 상태
- 서버 실행 상태
- 네트워크 상태

## 12. 운영 권장 사항

- 작업자 등록 시 이름과 담당 매듭 종류를 정확히 입력합니다.
- 공지와 가이드는 최신 작업 기준으로 정기 업데이트합니다.
- 신고 내역은 처리 후 상태를 꼭 `처리완료`로 변경합니다.
- 데모 계정은 운영 환경에서 별도 계정 체계로 교체하는 것을 권장합니다.
